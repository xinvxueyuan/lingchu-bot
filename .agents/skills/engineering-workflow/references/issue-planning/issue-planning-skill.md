# 议题规划

把“想法/问题/返工”压成可执行、可追踪的工作项。输出要能被工程实现继续使用。

## 路由

- 生成 PRD：读 `references/to-prd.md`。
- 拆成 issues：读 `references/to-issues.md`。
- triage 现有问题或反馈：读 `references/triage.md`。
- 写重构计划/RFC：读 `references/request-refactor-plan.md`。

## 规则

1. 先分清事实、假设、待确认问题和明确决策。
2. issue 要包含动机、范围、验收标准和验证建议。
3. 对 Lingchu Bot 的用户可见行为，记得同步 docs/tests/i18n/menu/trigger 的影响。
