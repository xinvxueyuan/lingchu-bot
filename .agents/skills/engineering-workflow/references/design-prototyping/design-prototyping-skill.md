# 设计原型

先帮用户把问题空间变清楚，再决定是提问、给方案，还是做可丢弃原型。

## 路由

- UI 或交互界面设计：读 `references/design-an-interface.md`。
- 设计 grill、质询需求、找隐含约束：读 `references/design-grill.md`。
- 需要快速验证方案的 throwaway prototype：读 `references/prototype.md`。

## 规则

1. 先识别目标用户、核心工作流、不可接受的失败模式。
2. 方案不确定时给 2-3 个真实不同的方向，并说明取舍。
3. 原型用于学习，不把一次性代码当最终架构。
4. 用户要求实现后，回到交付循环完成验证。
