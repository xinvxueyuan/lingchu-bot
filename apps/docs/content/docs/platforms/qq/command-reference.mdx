## QQ Commands

Commands are registered through `nonebot-plugin-alconna` with command start tokens and separators enabled by default. Command trigger words are locale-exclusive: Chinese locales enable Chinese triggers, English locales enable short hyphenated English triggers. The two trigger languages are not enabled at the same time.

## Menu

Primary command:

```text
иҸңеҚ•
```

English command when `LINGCHU_LOCALE` / `lc_locale` / `locale` selects English:

```text
menu
```

The menu command is an index entry. It lists the available submenu commands instead of expanding every feature into one long message. Submenus are code-driven and filtered by the selected platform, protocol, and implementation.

Default submenu commands:

- `member-management` / `жҲҗе‘ҳз®ЎзҗҶ`: kick, block/unblock, set member card, set member title, set/unset admin.
- `speech-management` / `еҸ‘иЁҖз®ЎзҗҶ`: mute/unmute member, enable/disable whole-group mute, recall recent messages.
- `group-chat-management` / `зҫӨиҒҠз®ЎзҗҶ`: set group name, set group avatar (when supported), send group announcement (when supported), leave current group.
- `remote-management` / `иҝңзЁӢз®ЎзҗҶ`: cross-group mute/unmute, whole-group mute/unmute, kick, block/unblock, send announcement.
- `system-management` / `зі»з»ҹз®ЎзҗҶ`: silence/speak (silent mode), boot/shutdown (handle gate).

<Callout type="info" title="Remote management">

`иҝңзЁӢз®ЎзҗҶ` / `remote-management` is a top-level menu entry that lists all remote (cross-group) commands. Remote commands accept a group ID (int) or group name (str). When a non-int string is provided, the bot falls back to fuzzy matching group names via the OneBot V11 `get_group_list` API.

</Callout>

## Member mute

```text
зҰҒиЁҖ @з”ЁжҲ· [ж—¶й•ҝз§’ж•°] [еҺҹеӣ ]
mute @user [duration seconds] [reason]
```

- Duration default: `60` seconds.
- Reason default: `иҝқеҸҚзҫӨи§„гҖҢй»ҳи®ӨгҖҚ`.

## Member unmute

```text
и§ЈзҰҒ @з”ЁжҲ·
unmute @user
```

<Callout type="info" title="Permission pre-checks and audit">

The unmute command runs permission pre-checks before executing: `check_target_privilege()` verifies the target user is not an admin/owner (unless the operator is the group owner or a superuser), and `check_bot_privilege()` verifies the bot has admin/owner role in the group. After execution, `record_command_audit()` writes a command-level audit log entry.

</Callout>

## Message recall

```text
ж’Өеӣһ [@з”ЁжҲ·] [ж•°йҮҸ]
recall [@user] [count]
```

- Count default: `recall_message_default_count` from `config.json5` (default `10`).
- A single numeric argument is treated as the count: `ж’Өеӣһ 20`.
- To target a user, mention them first: `ж’Өеӣһ @з”ЁжҲ· 20`.
- The command reads recent current-group message records from the message store, skips the triggering command message, verifies each candidate through OneBot V11 `get_msg`, then recalls it through `delete_msg`.
- The bot skips messages from the operator, itself, group admins/owner, and protected users configured through `protected_subject_feature_keys`.

## OneBot V11 blocklist

Blocklist and whitelist commands are available on OneBot V11 only. Blocking stores the user in the group or global blocklist, kicks the user from the current group, and rejects future group join requests while the block is active. Whitelisting stores the user as a protected subject; only repository-backed `SUPERUSERS` can add or remove whitelist entries.

```text
жӢүй»‘ @з”ЁжҲ· [ж—¶й•ҝз§’ж•°] [еҺҹеӣ ]
е…ЁеұҖжӢүй»‘ @з”ЁжҲ· [ж—¶й•ҝз§’ж•°] [еҺҹеӣ ]
еҲ й»‘ @з”ЁжҲ· [еҺҹеӣ ]
е…ЁеұҖеҲ й»‘ @з”ЁжҲ· [еҺҹеӣ ]
жё…з©әй»‘еҗҚеҚ• [еҺҹеӣ ]
е…ЁеұҖжё…з©әй»‘еҗҚеҚ• [reason]
жӢүзҷҪ @з”ЁжҲ· [еҺҹеӣ ]
е…ЁеұҖжӢүзҷҪ @з”ЁжҲ· [еҺҹеӣ ]
еҲ зҷҪ @з”ЁжҲ· [еҺҹеӣ ]
е…ЁеұҖеҲ зҷҪ @з”ЁжҲ· [еҺҹеӣ ]
```

English: `block`, `global-block`, `unblock`, `global-unblock`, `clear-blocklist`, `global-clear-blocklist`, `protect`, `global-protect`, `unprotect`, `global-unprotect`.

- Duration default: permanent.
- Block reason default: `иҝқеҸҚзҫӨи§„гҖҢй»ҳи®ӨгҖҚ`.
- Unblock/clear/whitelist reason default: `з®ЎзҗҶе‘ҳж“ҚдҪңгҖҢй»ҳи®ӨгҖҚ`.

`жё…з©әй»‘еҗҚеҚ•` only clears the current group blocklist. Use `е…ЁеұҖжё…з©әй»‘еҗҚеҚ•` to clear global entries.
`жӢүзҷҪ` and `еҲ зҷҪ` only affect the current group whitelist. Use the global variants for protected users across groups.

## Remote management (OneBot V11 only)

Remote commands operate on a **target group** other than the current one. The `<зҫӨеҸ·жҲ–зҫӨеҗҚз§°>` argument accepts:

- An integer group ID (e.g., `123456789`).
- A numeric string that can be parsed as an integer.
- A non-numeric string that triggers fuzzy matching against the bot's joined groups via `get_group_list`. Exact name match takes priority; otherwise, substring containment is used. Multiple matches cause the bot to ask for a more precise name or group ID.

| Capability | Chinese command | English command |
| --- | --- | --- |
| Remote mute | `иҝңзЁӢзҰҒиЁҖ <зҫӨеҸ·жҲ–зҫӨеҗҚз§°> @з”ЁжҲ· [ж—¶й•ҝз§’ж•°] [еҺҹеӣ ]` | `remote-mute <group_id_or_group_name> @user [duration seconds] [reason]` |
| Remote unmute | `иҝңзЁӢи§ЈзҰҒ <зҫӨеҸ·жҲ–зҫӨеҗҚз§°> @з”ЁжҲ· [еҺҹеӣ ]` | `remote-unmute <group_id_or_group_name> @user [reason]` |
| Remote whole mute | `иҝңзЁӢе…ЁдҪ“зҰҒиЁҖ <зҫӨеҸ·жҲ–зҫӨеҗҚз§°>` | `remote-mute-all <group_id_or_group_name>` |
| Remote whole unmute | `иҝңзЁӢе…ЁдҪ“и§ЈзҰҒ <зҫӨеҸ·жҲ–зҫӨеҗҚз§°>` | `remote-unmute-all <group_id_or_group_name>` |
| Remote kick | `иҝңзЁӢиёўеҮә <зҫӨеҸ·жҲ–зҫӨеҗҚз§°> @з”ЁжҲ· [еҺҹеӣ ]` | `remote-kick <group_id_or_group_name> @user [reason]` |
| Remote block | `иҝңзЁӢжӢүй»‘ <зҫӨеҸ·жҲ–зҫӨеҗҚз§°> @з”ЁжҲ· [ж—¶й•ҝз§’ж•°] [еҺҹеӣ ]` | `remote-block <group_id_or_group_name> @user [duration seconds] [reason]` |
| Remote unblock | `иҝңзЁӢеҲ й»‘ <зҫӨеҸ·жҲ–зҫӨеҗҚз§°> @з”ЁжҲ· [еҺҹеӣ ]` | `remote-unblock <group_id_or_group_name> @user [reason]` |

<Callout type="warn" title="Remote announcement availability">

Remote announcement requires `NapCat.Onebot >= 4.18.0`. Other OneBot V11 implementations cannot send group notices. The remote announcement menu entry is hidden when the implementation is unsupported.

</Callout>

<Callout type="info" title="Remote kick requires blocklist">

`иҝңзЁӢиёўеҮә` only works on users already in the blocklist for the target group. Use `иҝңзЁӢжӢүй»‘` first if the user is not yet blocked.

</Callout>

## Error feedback

Group management commands return readable error messages for covered adapter network errors and rejected operations. Unknown exceptions are still raised outward so developers can find uncovered cases.

Feedback text and command trigger language use `LINGCHU_LOCALE`, `lc_locale`, or `locale` from the current NoneBot configuration. See [Internationalization](../../developer-guide/engineering/i18n).

## Bot Control

```text
silence
й—ӯеҳҙ
```

Enable silent mode: commands still execute (mute, kick, etc. take effect) but no response messages are sent.

```text
speak
иҜҙиҜқ
```

Disable silent mode: resume sending command response messages.

```text
boot
ејҖжңә
```

Boot: enable all command handlers. All commands respond normally when booted.

```text
shutdown
е…іжңә
```

Shutdown: disable all command handlers (except boot/shutdown themselves). No commands execute or respond. Only boot/shutdown commands work when shut down.

<Callout type="warning" title="Shutdown Scope">

After shutdown, all group management commands (mute, kick, block, etc.) stop responding. Only the boot command can restore operation. In silent mode, commands still execute вҖ” only feedback messages are suppressed.

</Callout>
