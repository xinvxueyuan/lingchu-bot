## Lingchu Bot

![Lingchu Bot 文档概览](/images/docs/lingchu-bot-home.png)

Lingchu Bot 是一个基于 NoneBot2 的应用侧管理机器人项目。它以插件形式组织核心能力，面向群管理、命令处理、配置管理、本地存储和异步数据访问等场景。

<Callout type="warn" title="项目状态">

本项目仍处于 pre-alpha / development 阶段。当前代码、配置、命令行为和文档都可能继续调整，请不要把现有接口视为稳定生产接口。

</Callout>

## 当前能力

- 基于 NoneBot2 的核心插件入口。
- 通过 `pyproject.toml` 声明本地插件目录、可用适配器和 NoneBot 依赖插件。
- QQ 平台同一时间只启用一个适配器；默认选择 `~onebot.v11`，可用 `LINGCHUAdapter` 显式切换到 Milky。
- 当前已实现并测试覆盖的 QQ 群管理处理器：成员禁言/解禁、全体禁言/解禁、群资料设置、成员管理、群公告、踢人和退群。
- 可选消息存储：记录事件接收、处理状态、Bot 生命周期和平台 API 调用摘要，并把未知适配器归入 `unknown` 平台。
- JSON5 本地存储、NoneBot localstore 路径管理和 ORM CRUD 辅助能力。
- gettext / Babel 国际化辅助，当前包含 `zh_CN` 与 `en_US` 目录。
- 面向开发者的 prek 钩子、测试和 CI 检查流程。
- 面向后续迭代的跨平台和非群管理功能扩展空间。
