## Commands and Group Management

The currently confirmed user-facing capabilities are concentrated in QQ group management commands. Commands are registered through `nonebot-plugin-alconna`, with command start tokens and command separators enabled by default. The QQ platform uses OneBot V11 by default; set `LINGCHUAdapter = "~milky"` if you need Milky.

Success and failure feedback for group management commands is wired to gettext catalogs. Command trigger words are also selected from the configured locale: Chinese locales enable the existing Chinese triggers, while English locales enable short hyphenated English triggers. The two trigger languages are not enabled at the same time.

## Menu

Primary command:

```text
菜单
```

English command when `LINGCHU_LOCALE`/`lc_locale`/`locale` selects English:

```text
menu
```

Common aliases include:

- `帮助`
- `功能`
- `功能列表`
- `指令`
- `命令列表`
- `help`
- `commands`

The menu command lists the currently registered group-management features as plain text. The default menu is grouped into multiple levels:

- Member Management: kick member, set member card, set member special title, set/unset admin.
- Speech Management: mute/unmute member, enable/disable whole-group mute.
- Group Profile: set group name and avatar.
- Announcement: send group announcement.
- Group Operation: leave current group.

On first startup, Lingchu creates `menu.json5` through `nonebot-plugin-localstore`'s plugin config directory. Existing files are not overwritten, so operators can edit titles, summaries, usages, and sections. Menu items reference the internal command keys from the trigger catalog; unknown keys are skipped when rendering instead of breaking the command.

## Member mute

Primary command:

```text
禁言 @用户 [duration in seconds] [reason]
```

English command when `LINGCHU_LOCALE`/`lc_locale`/`locale` selects English:

```text
mute @user [duration in seconds] [reason]
```

Default values:

- Duration: `60` seconds.
- Reason: `违反群规「默认」`.

## Group settings and operations

These commands are the QQ group management handlers registered in the current source. Commands act on the current group by default, and `@用户` resolves to the target group member. The plugin enables only one QQ adapter per platform selection; the table does not list future capabilities as currently available commands.

| Capability | Chinese command | English command |
| --- | --- | --- |
| Set group name | `设置群名称 <new group name>` | `set-group-name <new group name>` |
| Set group avatar | `设置群头像 <image_uri>` | `set-group-avatar <image_uri>` |
| Set member card | `设置群名片 @用户 <card>` | `set-member-card @user <card>` |
| Set member special title | `设置群头衔 @用户 <title>` | `set-member-title @user <title>` |
| Set member admin | `设置群管理员 @用户 [true/false]` | `set-member-admin @user [true/false]` |
| Unset member admin | `取消群管理员 @用户` | `unset-member-admin @user` |
| Kick group member | `踢出群成员 @用户 [reject add request]` | `kick-member @user [reject add request]` |
| Send announcement | `发送群公告 <content> [image_uri]` | `send-announcement <content> [image_uri]` |
| Leave current group | `退出群` | `leave-group` |

Representative English aliases include `rename-group`, `change-group-avatar`, `set-group-card`, `set-special-title`, `set-admin`, `promote-admin`, `unset-admin`, `revoke-admin`, `kick`, `remove-member`, `announce`, `group-announcement`, and `quit-group`.

The table lists both language sets for reference. Runtime command registration uses only the language selected by the configured locale.

The announcement and group avatar commands accept `UniImage` values from `nonebot-plugin-alconna`. The current implementation resolves raw image bytes, local paths, or downloadable URLs into cached files before passing them to adapter APIs. Quote names, announcements, cards, or titles that contain spaces so command separators do not split them.

Essence message management and similar capabilities are candidates for later iterations. They are not registered user-triggered commands in the current source.

## Error feedback

Group management commands return readable error messages for covered adapter network errors and rejected operations. Unknown exceptions are still raised outward so developers can find uncovered cases.

Feedback text and command trigger language use `LINGCHU_LOCALE`, `lc_locale`, or `locale` from the current NoneBot configuration. Without configuration, the default follows the i18n helper default locale. The current repository provides `zh_CN` and `en_US` catalogs. See [Internationalization](../developer-guide/i18n).
