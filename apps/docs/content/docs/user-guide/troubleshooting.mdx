## Troubleshooting

This page collects the most common local startup and group management issues currently covered by the project.

## Dependency installation fails

First confirm that Python and uv are available:

```bash
python --version
uv --version
```

The project requires Python 3.13. Dependency installation should use:

```bash
uv sync --frozen
```

## Boolean configuration errors

If you see an `in_containers 配置错误` error, check `.env` or the NoneBot configuration file.

Correct:

```dotenv
IN_CONTAINERS=true
```

Incorrect:

```dotenv
IN_CONTAINERS=True
```

## Bot cannot connect to the platform

Check these first:

- Whether the service for the enabled adapter has started; the default is OneBot V11.
- Whether NoneBot and the enabled adapter connection settings match.
- Whether the account, network, firewall, and platform permissions are usable.

## Adapter selection and loading

Lingchu Bot only selects business code through `LINGCHUAdapter`; it does not control which adapters NoneBot actually loads.

- When `LINGCHUAdapter` is unset, Lingchu Bot enables `~onebot.v11` by default; make sure NoneBot has loaded OneBot V11.
- To pin OneBot V11, set `LINGCHUAdapter = "~onebot.v11"` and load the OneBot V11 adapter.
- To pin Milky, set `LINGCHUAdapter = "~milky"` and load the Milky adapter.
- Do not explicitly configure multiple adapters for the same platform. For example, `LINGCHUAdapter = "~milky+~onebot.v11"` fails immediately.
- Do not explicitly declare adapters Lingchu has not implemented or cannot recognize in `LINGCHUAdapter`; extra unknown adapters registered at runtime are ignored.
- If startup reports `PlatformAdapterNotLoadedError`, the selected adapter was not loaded or registered by NoneBot; load the matching adapter or change `LINGCHUAdapter` to one that is already loaded.
- Extra registered same-platform adapters do not create a conflict. Lingchu Bot treats unselected adapters as disabled.

## Group management command fails

Mute, unmute, and whole-group mute depend on platform permissions. When they fail, check:

- Whether the bot is in the target group.
- Whether the bot has the required management permission.
- Whether the target user can be operated on by the bot.
- Whether the enabled adapter API reports a network error or rejected operation.

If feedback appears in an unexpected language, check whether `LINGCHU_LOCALE`, `lc_locale`, or `locale` is set to a supported catalog such as `zh_CN` or `en_US`. Unknown text falls back to the original message.

## CI or local checks fail

Read the failing command and line number, then fix only the related scope. Common checks include:

```bash
uv run -m ruff check . --output-format=github
uv run -m ruff format --check .
uv run -m pyright .
uv run -m ty check --output-format github
uv run -m pytest
```
