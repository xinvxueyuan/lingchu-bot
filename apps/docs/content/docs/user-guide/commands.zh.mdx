## 命令与群管理

当前可确认的用户侧能力集中在 QQ 群管理命令。命令由 `nonebot-plugin-alconna` 注册，默认启用命令起始符和命令分隔符。QQ 平台默认使用 OneBot V11；如果需要使用 Milky，请在配置中设置 `LINGCHUAdapter = "~milky"`。

群管理命令的成功和失败反馈已经接入 gettext 翻译目录。命令触发词也会按配置的 locale 选择：中文 locale 启用既有中文触发词，英文 locale 启用短横线风格英文触发词。两种语言的触发词不会同时启用。

## 菜单

主命令：

```text
菜单
```

当 `LINGCHU_LOCALE` / `lc_locale` / `locale` 选择英文时的英文命令：

```text
menu
```

常见别名包括：

- `帮助`
- `功能`
- `功能列表`
- `指令`
- `命令列表`
- `help`
- `commands`

菜单命令会以纯文本列出当前已注册的群管理功能。默认菜单使用多级分类：

- 成员管理：踢出群成员、设置群名片、设置群头衔、设置/取消管理员。
- 发言管理：禁言/解禁群成员、开启/关闭全体禁言。
- 群资料：设置群名称、设置群头像。
- 群公告：发送群公告。
- 群操作：退出当前群。

首次启动时，Lingchu 会通过 `nonebot-plugin-localstore` 的插件配置目录创建 `menu.json5`。已有文件不会被覆盖，因此运营者可以编辑标题、摘要、用法和分组。菜单项通过触发词目录中的内部命令 key 关联真实命令；未知 key 会在渲染时跳过，不会让菜单命令失败。

## 成员禁言

主命令：

```text
禁言 @用户 [时长秒数] [原因]
```

当 `LINGCHU_LOCALE` / `lc_locale` / `locale` 选择英文时的英文命令：

```text
mute @user [时长秒数] [原因]
```

默认值：

- 时长：`60` 秒。
- 原因：`违反群规「默认」`。

常见别名包括：

- `禁言用户`
- `禁言群成员`
- `禁言成员`
- `ban`
- `mute-member`
- `ban-member`
- `禁`
- `封禁`

## 群设置与操作

以下命令是当前源码中已经注册的 QQ 群管理处理器。命令默认作用于当前群，`@用户` 会解析为目标群成员。插件会按平台只启用一个 QQ 适配器；下表不把尚未实现的未来能力列为已可用命令。

| 能力 | 中文命令 | 英文命令 |
| --- | --- | --- |
| 设置群名称 | `设置群名称 <新群名称>` | `set-group-name <new group name>` |
| 设置群头像 | `设置群头像 <image_uri>` | `set-group-avatar <image_uri>` |
| 设置群名片 | `设置群名片 @用户 <名片>` | `set-member-card @user <card>` |
| 设置群成员专属头衔 | `设置群头衔 @用户 <头衔>` | `set-member-title @user <title>` |
| 设置群管理员 | `设置群管理员 @用户 [true/false]` | `set-member-admin @user [true/false]` |
| 取消群管理员 | `取消群管理员 @用户` | `unset-member-admin @user` |
| 踢出群成员 | `踢出群成员 @用户 [是否拒绝再次申请]` | `kick-member @user [reject add request]` |
| 发送群公告 | `发送群公告 <内容> [image_uri]` | `send-announcement <content> [image_uri]` |
| 退出当前群 | `退出群` | `leave-group` |

代表性英文别名包括 `rename-group`、`change-group-avatar`、`set-group-card`、`set-special-title`、`set-admin`、`promote-admin`、`unset-admin`、`revoke-admin`、`kick`、`remove-member`、`announce`、`group-announcement` 和 `quit-group`。

上表同时列出两套语言用于查阅；运行时只会注册当前 locale 选中的那一套命令触发词。

群公告和群头像命令接收 `nonebot-plugin-alconna` 的 `UniImage` 参数。当前实现会把图片原始字节、本地路径或可下载 URL 解析到缓存文件，再交给适配器 API。包含空格的名称、公告内容、名片或头衔建议使用引号包裹，避免被命令分隔符拆分。

群精华消息等能力属于后续迭代候选项，当前没有注册为用户可触发命令。

## 错误反馈

群管理命令会对已覆盖适配器的网络异常和操作拒绝返回可读错误消息。未知异常仍会向外抛出，方便开发者发现未覆盖的问题。

反馈文本和命令触发词语言会使用当前 NoneBot 配置中的 `LINGCHU_LOCALE`、`lc_locale` 或 `locale`。未配置时遵循 i18n 辅助函数的默认 locale，当前仓库提供 `zh_CN` 和 `en_US` catalog。详见 [国际化](../developer-guide/i18n)。
