## 命令与群管理

当前可确认的用户侧能力集中在 QQ 群管理命令。命令由 `nonebot-plugin-alconna` 注册，默认启用命令起始符和命令分隔符。QQ 平台默认使用 OneBot V11；如果需要使用 Milky，请在配置中设置 `LINGCHUAdapter = "~milky"`。

群管理命令的成功和失败反馈已经接入 gettext 翻译目录。命令触发词仍以当前中文命令和别名为主，切换 locale 不会自动新增英文命令触发词。

## 成员禁言

主命令：

```text
禁言 @用户 [时长秒数] [原因]
```

默认值：

- 时长：`60` 秒。
- 原因：`违反群规「默认」`。

常见别名包括：

- `禁言用户`
- `禁言群成员`
- `禁言成员`
- `禁`
- `封禁`

## 成员解禁

## 群设置与操作

以下命令是当前源码中已经注册的 QQ 群管理处理器。命令默认作用于当前群，`@用户` 会解析为目标群成员。插件会按平台只启用一个 QQ 适配器；下表不把尚未实现的未来能力列为已可用命令。

| 能力 | 主命令 |
| --- | --- |
| 设置群名称 | `设置群名称 <新群名称>` |
| 设置群头像 | `设置群头像 <image_uri>` |
| 设置群名片 | `设置群名片 @用户 <名片>` |
| 设置群成员专属头衔 | `设置群头衔 @用户 <头衔>` |
| 设置群管理员 | `设置群管理员 @用户 [true/false]` |
| 取消群管理员 | `取消群管理员 @用户` |
| 踢出群成员 | `踢出群成员 @用户 [是否拒绝再次申请]` |
| 发送群公告 | `发送群公告 <内容> [image_uri]` |
| 退出当前群 | `退出群` |

群公告和群头像命令接收 `nonebot-plugin-alconna` 的 `UniImage` 参数。当前实现会把图片原始字节、本地路径或可下载 URL 解析到缓存文件，再交给适配器 API。包含空格的名称、公告内容、名片或头衔建议使用引号包裹，避免被命令分隔符拆分。

群精华消息等能力属于后续迭代候选项，当前没有注册为用户可触发命令。

## 错误反馈

群管理命令会对已覆盖适配器的网络异常和操作拒绝返回可读错误消息。未知异常仍会向外抛出，方便开发者发现未覆盖的问题。

反馈文本会使用当前 NoneBot 配置中的 `LINGCHU_LOCALE`、`lc_locale` 或 `locale`。未配置时默认使用 `zh_CN`，当前仓库提供 `zh_CN` 和 `en_US` catalog。详见 [国际化](../developer-guide/i18n)。
