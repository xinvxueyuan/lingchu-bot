## 命令概览

![Lingchu Bot 命令与群管理](/images/docs/commands-management.webp)

Lingchu Bot 的用户侧能力集中在 QQ 群管理命令。命令由 `nonebot-plugin-alconna` 注册，默认启用命令起始符和命令分隔符。

## Locale 与触发词

命令触发词按 locale 互斥选择。中文 locale 启用既有中文触发词，英文 locale 启用短横线风格英文触发词。两种语言的触发词不会同时启用。

反馈文本和命令触发词语言会使用当前 NoneBot 配置中的 `LINGCHU_LOCALE`、`lc_locale` 或 `locale`。详见[国际化](../developer-guide/i18n)。

## 菜单

主命令：

```text
菜单
```

当 `LINGCHU_LOCALE` / `lc_locale` / `locale` 选择英文时的英文命令：

```text
menu
```

菜单命令是索引入口。它只列出可进入的子菜单命令，不再把所有功能堆在一条长消息里。子菜单完全由代码驱动，并按当前平台、协议和实现过滤。

默认子菜单命令：

- `成员管理` / `member-management`
- `发言管理` / `speech-management`
- `群聊管理` / `group-chat-management`
- `远程管理` / `remote-management`
- `机器人控制` / `bot-control`

## 实现过滤

不被当前平台、协议或实现支持的命令会从菜单中隐藏。当前实现过滤规则包括：

- OneBot V11 未知实现会隐藏群公告和群头像。
- `LLOneBot >= 7.12.0` 显示群公告，但隐藏群头像。
- `NapCat.Onebot >= 4.18.0` 显示群公告和群头像。
- Milky 未知实现会隐藏群公告。
- Milky `LLBot` 显示纯文本群公告；菜单不会宣称图片公告用法。
- 远程管理命令仅 OneBot V11 可用。
- 远程公告要求 `LLOneBot >= 7.12.0` 或 `NapCat.Onebot >= 4.18.0`。

## 错误反馈

群管理命令会对已覆盖适配器的网络异常和操作拒绝返回可读错误消息。未知异常仍会向外抛出，方便开发者发现未覆盖的问题。
