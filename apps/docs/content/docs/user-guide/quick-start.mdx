## Quick Start

This page explains how to prepare Lingchu Bot locally. Before connecting to a real platform, prepare the account, network, and configuration required by NoneBot and the enabled adapter. The QQ platform enables OneBot V11 by default; configure Milky explicitly if you need it.

## Requirements

- Python 3.13
- uv
- git
- A runtime environment usable by the enabled QQ adapter; OneBot V11 is the default

## Get the code

```bash
git clone https://github.com/xinvxueyuan/lingchu-bot.git
cd lingchu-bot
```

## Install dependencies

```bash
uv sync --frozen
```

## Start

The current working tree does not ship a committed root-level `bot.py`. Lingchu Bot is primarily a NoneBot plugin package, and `[tool.nonebot]` in `pyproject.toml` declares `plugin_dirs = ["src/plugins"]`.

You can load this repository as a local plugin directory from an existing NoneBot project, or run it through the Docker build flow provided by the repository. The Docker image generates the runtime `/tmp/bot.py` with `nb-cli` during build.

```bash
docker compose up --build
```

## Pre-run checklist

- Confirm that Python is version 3.13.
- Confirm that the service and connection settings for the enabled adapter are ready.
- If runtime registers multiple QQ adapters, Lingchu Bot only enables the business code selected by `LINGCHUAdapter`; make sure the selected adapter has also been loaded by NoneBot.
- Confirm that the target NoneBot project loads `src/plugins`, or that you are using the repository Docker runner.
- Confirm that Boolean values in NoneBot configuration use lowercase `true` / `false`, not `True` / `False`.
