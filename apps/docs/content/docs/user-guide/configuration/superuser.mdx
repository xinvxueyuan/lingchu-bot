title: Superuser Identity
---

Lingchu permissions use stable UIDs and platform account bindings instead of treating a platform account ID as the only identity. Configure the dedicated superuser mapping as strict JSON inside the environment variable:

```dotenv
LINGCHU_SUPERUSERS='{"userA":{"qq":123,"telegram":"tg-id","discord":"discord-id"}}'
```

The object key is the Lingchu UID. Each value maps platform IDs, such as `qq`, `telegram`, or `discord`, to that user's account ID on the platform.

`LINGCHU_SUPERUSERS` is required. Startup fails when it is missing or empty, the value is not a UID-to-platform mapping, a platform account is bound to multiple UIDs, or an unknown platform ID is used.

SUPERUSERS always pass Lingchu command permission checks, but platform API restrictions and command-level safety checks still apply.

See [Environment Variables](./environment-variables) for the env var reference, or [Configuration](/docs/user-guide/configuration) for runtime settings.
