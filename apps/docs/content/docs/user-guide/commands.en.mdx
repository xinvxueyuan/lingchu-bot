## Commands and Group Management

The currently confirmed user-facing capabilities are concentrated in Milky group management commands. Commands are registered through `nonebot-plugin-alconna`, with command start tokens and command separators enabled by default.

Success and failure feedback for group management commands is wired to gettext catalogs. Command trigger words are still primarily the current Chinese commands and aliases. Switching locale does not automatically add English command trigger words.

## Member mute

Primary command:

```text
禁言 @用户 [duration in seconds] [reason]
```

Default values:

- Duration: `60` seconds.
- Reason: `违反群规「默认」`.

Common aliases include:

- `禁言用户`
- `禁言群成员`
- `禁言成员`
- `禁`
- `封禁`

## Member unmute

## Group settings and operations

These commands are the Milky group management handlers registered in the current source. Commands act on the current group by default, and `@用户` resolves to the target group member. Plugin metadata and project configuration preserve room for multi-adapter expansion, but the table does not list future capabilities as currently available commands.

| Capability | Primary command |
| --- | --- |
| Set group name | `设置群名称 <new group name>` |
| Set group avatar | `设置群头像 <image_uri>` |
| Set member card | `设置群名片 @用户 <card>` |
| Set member special title | `设置群头衔 @用户 <title>` |
| Set member admin | `设置群管理员 @用户 [true/false]` |
| Unset member admin | `取消群管理员 @用户` |
| Kick group member | `踢出群成员 @用户 [reject add request]` |
| Send announcement | `发送群公告 <content> [image_uri]` |
| Leave current group | `退出群` |

The announcement and group avatar commands accept `UniImage` values from `nonebot-plugin-alconna`. The current implementation resolves raw image bytes, local paths, or downloadable URLs into cached files before passing them to adapter APIs. Quote names, announcements, cards, or titles that contain spaces so command separators do not split them.

Essence message management and similar capabilities are candidates for later iterations. They are not registered user-triggered commands in the current source.

## Error feedback

Group management commands return readable error messages for Milky network errors and rejected operations. Unknown exceptions are still raised outward so developers can find uncovered cases.

Feedback text uses `LINGCHU_LOCALE`, `lc_locale`, or `locale` from the current NoneBot configuration. Without configuration, the default is `zh_CN`. The current repository provides `zh_CN` and `en_US` catalogs. See [Internationalization](../developer-guide/i18n).
